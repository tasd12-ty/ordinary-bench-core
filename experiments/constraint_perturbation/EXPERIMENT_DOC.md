# QRR 约束图的最大无环翻转容量实验

## 实验目的

回答一个核心问题：**在一个场景的距离序约束中，最多能翻转多少个约束而不产生逻辑矛盾？**

这个数字决定了"自洽犯错空间"的大小——如果 VLM 的错误率低于这个容错上限，但 reconstruction 仍然失败，说明模型的错误包含逻辑矛盾，不是"答错太多"，而是"答案自相矛盾"。

## 背景知识

### QRR 约束

N 个物体产生 C(N,2) 个距离对，两两比较得到 C(C(N,2), 2) 个约束：

```
d(A,B) < d(C,D)    A-B 距离比 C-D 更近
d(A,B) > d(C,D)    A-B 距离比 C-D 更远
d(A,B) ~= d(C,D)   约等于（在 τ=0.10 容差内）
```

其中 `<` 和 `>` 称为 **strict 约束**，`~=` 称为 **approx 约束**。

### DAG 环（逻辑矛盾）

距离大小关系具有传递性：`d(A,B) < d(C,D)` 且 `d(C,D) < d(E,F)` 推出 `d(A,B) < d(E,F)`。如果约束集中同时存在 `d(A,B) < ... < d(E,F)` 和 `d(A,B) > d(E,F)`，就形成了 DAG 中的**环**——不存在任何物体摆放方式能同时满足这些约束。

### 两种扰动方式

| 扰动方式 | 操作 | 是否可能产生环 | 含义 |
|---------|------|-------------|------|
| **Flip**（翻转） | `<` ↔ `>` | 是（反转 DAG 有向边） | 模型把大小关系搞反 |
| **Relax**（松弛） | `<` 或 `>` → `~=` | 否（删除 DAG 边，永远安全） | 模型把明确关系模糊化 |

## 翻转比例上限的计算方法

### 问题定义

给定一组 QRR 约束（构成一个 DAG），求最多能翻转多少个 strict 约束（`<` ↔ `>`）使得翻转后的约束集仍然是 DAG（无环）。

**这是一个 NP-hard 问题**——等价于最小反馈弧集（Minimum Feedback Arc Set）的对偶问题，没有已知的多项式时间精确算法。

### 贪心近似算法

我们使用贪心策略求下界：

```
算法：consistent_flip_qrr(constraints, fraction=1.0, rng)

输入: QRR 约束列表, 随机种子
输出: (扰动后约束列表, 实际翻转数)

1. 筛选所有 strict 约束索引（< 或 >，排除 ~=）
2. 按随机种子打乱候选顺序
3. 逐个尝试翻转:
   a. 将当前约束 < 改为 >（或 > 改为 <）
   b. 对整个约束列表构建距离偏序 DAG
   c. 用 DFS 着色法检测环 → O(V+E)
   d. 有环 → 撤回翻转，跳过该约束
   e. 无环 → 保留翻转，计数 +1
4. 遍历完所有候选，返回成功翻转总数
```

**时间复杂度**: O(K × (V+E))，其中 K = strict 约束数，V = 距离对数（DAG 节点），E = strict 约束数（DAG 边）。

### 种子依赖性

贪心结果依赖遍历顺序——先翻转的约束可能"占据"某些传递链位置，阻止后续翻转。不同随机种子给出不同结果。

- N=4~10 场景：每个场景 10 个种子，取最大值
- N=11~15 场景：每个场景 1 个种子（大约束图较慢）

贪心给出的是**下界**——真正的最大无环翻转数可能更高。

### Relax 的容量

Relax（`<` 或 `>` → `~=`）等于删除 DAG 边。删边永远不产生环，因此 **relax 容量 = strict 约束总数（100%）**。无需算法测试。

### 代码位置

- 翻转算法: `perturbation.py` → `consistent_flip_qrr()`
- 环检测: `VLM-test/reconstruct/constraints.py` → `build_distance_poset()` → DFS 着色法

### 关键发现

1. **N=5~7 约束图最刚性**：平均只有 6-7% 的 strict 约束可以无环翻转。VLM 典型错误率 20-30% 远超此上限 → 错误必然含矛盾。

2. **容错率随 N 增大而上升**：从 N=5 的 7% 升至 N=15 的 69%。大约束图中很多约束相互独立，翻转不影响其他。

3. **场景间差异巨大**：同一 N，有的场景 0 个都翻不了（完全刚性 DAG），有的可翻 70%+。取决于物体的具体几何布局。

4. **N=14 出现 95% 极端值**：一个场景 3460 个 strict 约束中可翻转 3279 个，接近全部。

## 与 VLM 错误率的对比（核心论证）

### 论证逻辑

1. **Null Model**: 如果 VLM 的错误是"自洽的"（翻错但不矛盾），只要错误率 < 翻转容错上限，reconstruction 应该 feasible
2. **对比**: 实际 VLM 错误率 vs 翻转容错上限
3. **结论**: 如果错误率 < 容错上限但 reconstruction 失败 → 错误包含逻辑矛盾 → 模型缺乏一致空间表征

### 结果（N=4~10，有 VLM 数据）

图表: `results/fig_vlm_vs_tolerance.png`

**N=5~8 区间最关键**：翻转容错 6-13%，所有 VLM 错误率 20-47%。差距 3-7 倍——VLM 的错误量远超自洽容错空间，必然含逻辑矛盾。

**N=10**：翻转容错升至 31%，部分模型（Gemini 22%、Qwen 25%）的错误率落入容错区间。如果这些模型在 N=10 的 reconstruction 失败，说明错误虽然在数量上可自洽，但结构上仍含矛盾。

**N=11~15 预测**：容错率 39-69%，超过大部分 VLM 的错误率。如果 VLM 在这些大场景上 reconstruction 仍然失败，将是更强的论据——即使约束图"松"到允许 60%+ 的错误，模型的错误结构仍然产生矛盾。
