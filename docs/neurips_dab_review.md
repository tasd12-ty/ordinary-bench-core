## 2. NeurIPS D&B Track 适配性评估

### 2.1 Track 核心要求 vs 当前工作

| D&B 要求 | 当前状态 | 评分 |
|---------|--------|------|
| **新颖的数据集/基准** | 700 场景、327K 探测点、受控 Blender 渲染 | ✅ 强 |
| **清晰的数据文档** | 场景 JSON + 问题生成代码 + GT 标注 | ✅ 强 |
| **基线结果** | 2 个模型 (Qwen-235B, Qwen-30B)，仅 ~60 场景 | ⚠️ 不足 |
| **多模型覆盖** | 仅 Qwen 系列 | ❌ 严重不足 |
| **人类基线** | 规划有 (§4.6.3) 但未实现 | ❌ 缺失 |
| **失败模式分析** | 三级错误归因框架已设计 | ⚠️ 需数据支撑 |
| **更广泛影响** | 神经科学行为反演视角有新意 | ✅ 强 |
| **可复现性** | 完整代码 + 合成数据 = 完全可复现 | ✅ 强 |

### 2.2 核心卖点（对 D&B 审稿人）

**卖点 1：「解释性基准」而非「又一个准确率排行榜」**

- 关键创新：不止测对错，而是从序数判断逆向重建 VLM 的内部场景表征
- 这让 benchmark 提供的信息量远超传统 accuracy-based 基准
- H1 直接可检验："单题准确率高估 VLM 的几何能力"

**卖点 2：三条件实验是干净的消融设计**

- 正确图像 / 错误图像 / 无图像 → 分离视觉贡献 vs 语言先验
- 对 D&B 审稿人来说，这不是可选的 ablation，而是 benchmark 的核心协议
- H3 直接可检验

**卖点 3：受控合成场景是 feature 不是 bug**

- 精确 GT 坐标、可控复杂度梯度 (n=4→10)、无标注噪声
- 类似 CLEVR 对 VQA 的贡献——牺牲生态效度换取实验控制力

### 2.3 主要风险与短板

**❌ Risk 1: 模型覆盖太窄**

- D&B track 期望看到 5-8+ 个多样化模型
- 当前仅 2 个 Qwen 模型，审稿人会质疑 benchmark 的通用性
- **必须补充**: GPT-4o, Gemini 2.0 Flash, Claude Sonnet, LLaVA-OneVision, InternVL2 等

**❌ Risk 2: 无人类基线**

- D&B track 几乎必须有人类表现参照
- PDF 规划稿已提到 (§4.6.3)，但未实现
- **必须补充**: 至少 50-100 场景的人类标注

**⚠️ Risk 3: 重建被误读为「方法论文」**

- Scene belief reconstruction 是很强的贡献，但如果写法偏向方法论文，D&B 审稿人可能说"这不属于我们 track"
- **解法**: 将重建定位为 "benchmark 提供的分析工具"，而非独立的方法贡献
- 类比：ImageNet 论文本身不是算法论文，但提供了评估框架

**⚠️ Risk 4: 动态/时间模块不成熟**

- PDF 规划稿已正确处理：写成「扩展主实验」而非核心贡献
- 但如果数据不足，不如砍掉或纯放附录
- 审稿人对"半成品"模块容忍度很低

**⚠️ Risk 5: 训练可塑性偏离 D&B 定位**

- GRPO/SFT 训练实验更像 ML 研究而非 benchmark 贡献
- **建议**: v1 完全砍掉训练部分，或缩到一段作为"benchmark 用途示例"

### 3.1 结构调整（面向 D&B track）

**推荐正文结构 (9 页)：**

| § | 标题 | 页数 | 说明 |

| 1 | Introduction | 1.0 | 核心缺口 → 行为反演思路 → 贡献列表 |

- 砍掉 §6 (时间知觉) 从正文移到附录

- 合并 §3 (问题定义) 到 §3+§4

## 5. 总体判断

### 适合 NeurIPS D&B Track 吗？

**潜力评级：7/10（有条件适合）**

**适合的理由：**

- 「解释性基准」角度在 D&B track 中罕见且有价值
- 受控合成场景 + 完整可复现代码是 D&B 的加分项
- Scene belief reconstruction 作为分析工具是真正的新贡献
- 三条件实验设计干净，信息量大

**需要补齐才能投的硬性缺口：**

1. 模型覆盖 (2 → 6+ 模型)
2. 人类基线
3. 全量场景结果 (60 → 700 场景)
4. 三条件实验数据

**投稿前需要做的写法调整：**

1. 重建定位为「benchmark 分析工具」而非独立方法
2. 训练可塑性砍掉或极度压缩
3. 时间模块压缩到附录或短段
4. 标题应突出 benchmark 性质，建议回归 PDF 版的「行为反演」框架

### 备选投稿策略

| 目标 | 策略 | 风险 |
|------|------|------|
| **NeurIPS D&B** (首选) | 聚焦 benchmark + 重建分析工具 | 需补模型+人类基线 |
| **ICLR 2026 主会** | 强调重建方法 + 理论贡献 | 需要更深的理论分析 |
| **CVPR/ECCV** | 强调视觉感知评估 | 与 SpatialVLM 等竞争激烈 |
| **ACL/EMNLP** | 强调语言-视觉分离 | 审稿人可能不熟悉重建 |
